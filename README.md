## Archon

I build tools for thinking with AI, and write about the structure underneath markets, machine learning, and human behavior — the part most explanations skip.

### Working in the open

- **[claude-md-interviewer](https://github.com/archon-kb/claude-md-interviewer)** — a Claude Code skill that writes a project's `CLAUDE.md` by interviewing you, instead of you guessing what to put there. One idea underneath it: an instruction file is the agent's *character*, not its memory — and attention is zero-sum, so the few always-true rules earn their place and nothing else gets in.

### Math as the foundation

Finance, banking, and AI are applied mathematics wearing domain clothes. I keep the math as the load-bearing layer, not the decoration — and I'm building toward a tool that makes that layer learnable by anyone, the way a good tutor makes it obvious in hindsight.

### How I work

Human-in-the-loop, on purpose. The quality comes from the pairing — a person who insists on understanding every part, and an AI that does the reach. Not automation that replaces the thinking; leverage that sharpens it.

**Writing:** [Под продуктивностью](https://t.me/archon_KB) — how AI mechanisms actually work, taken mechanism-first instead of trick-first.

### RU

Строю инструменты для мышления с ИИ и пишу о структуре под рынками, машинным обучением и поведением людей — той части, которую большинство объяснений пропускает.

- **[claude-md-interviewer](https://github.com/archon-kb/claude-md-interviewer)** — скилл Claude Code, который собирает `CLAUDE.md` проекта через интервью, а не угадывание. Идея одна: файл инструкций — это *характер* агента, не его память; внимание — ресурс с нулевой суммой, поэтому в файл попадают только немногие всегда-истинные правила, и больше ничего.
- **Математика как фундамент** — финансы, банкинг и ИИ это прикладная математика в одежде предметной области. Держу математику несущим слоем, а не украшением, и строю инструмент, который делает этот слой доступным каждому.
- **Human-in-the-loop осознанно** — качество приходит из связки человек+ИИ, не из автономии.
